# v3 — Name it, and place the portfolio (2026-06-10)

The final calls: what to name the build hub, and where the freelancer portfolio lives.

```
My long-term vision is for SMOALD to become a holding company, but right now
I'm a freelancer trying to find work. Should my portfolio go under the SMOALD
parent, or be a separate site? And should the build hub be SMOALD Tech, Dev,
or AI?
```

**What changed:**
- **Naming** — "Dev" undersells and "Tech" is generic; the build hub became **SMOALD AI**, matching my actual focus and filtering for higher-value clients.
- **Positioning** — build the founder brand that gets me hired *now*, not the full holding company; grow into that structure once the divisions genuinely exist.
- **Portfolio** — it lives *under* SMOALD (later wired to `/ai/hire`), so clients meet a founder, not a freelancer asking for a favour.

This three-hub model became the brief the [`build-smoald-brand-hub`](../../build-smoald-brand-hub/) prompt turned into the live site.
